# Claude 自动化/机器人/智能制造子方向独立审核

审核时间：2026-05-09

审核对象：自动化类/机器人工程/智能制造，面向中国大陆 2027-2031 本科/硕士入口岗位。

## Claude 审核摘要

A 必须修改的评级或分类

1. 控制算法/系统辨识/MPC：必须从中高降为中。物理对象建模、鲁棒性保障和安全约束是 AI 最难穿透的壁垒；工业现场的控制岗壁垒高于仿真场景。
2. AMR/AGV/仓储物流自动化：建议从中高升为高，或至少标记为临界。ROS、Gazebo、开源 SLAM 和导航栈使入门 demo 极其模板化。
3. PLC/HMI/SCADA 基础集成：临界，有升为高的压力。样板程序、组态和点表生成已被 LLM 显著提效，但现场联调仍形成约束。

B 明显遗漏的子方向

1. 工业机器人示教/离线编程/轨迹规划：示教再现、离线编程和简单轨迹生成是独立入门工种，评级中高。
2. 自动化系统测试执行/工厂验收测试执行：重复执行测试用例、记录数据、生成报告高度流程化，评级高。

C 最终建议

建议分布调整为 3 高 / 5 中高 / 7 中 / 3 低 / 3 机会，或在保留原 20 项时调整为 3 高 / 4 中高 / 7 中 / 3 低 / 3 机会。总判断是：控制算法评级是唯一硬伤，AMR 和 PLC 两项低估了入门岗的模板化程度；整体框架对物理现场属性的权重分配基本正确，但仿真、论文、开源栈对入门岗的替代速度在多个条目中被系统性低估。

## 采纳结果

- 采纳“控制算法降为中”。
- 采纳“AMR/AGV 升为高”。
- 采纳“工业机器人示教/离线编程/轨迹规划”新增为中高。
- 采纳“自动化系统测试执行/FAT/SAT 报告”新增为高。
- PLC/HMI/SCADA 保留中高，不升为高；理由是大陆大量岗位仍带有现场联调、设备通讯和安全联锁责任，但在文案中标明临界压力。
- 最终使用 22 项版本，分布为：4 高、5 中高、7 中、3 低、3 机会。
